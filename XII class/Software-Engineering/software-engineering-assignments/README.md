# Софтуерно инженерство – XII клас
## Поредица от задания върху дипломния проект

Това repository съдържа заданията по предмет **„Софтуерно инженерство“** за ученици от XII клас, специалност **„Приложен програмист“**.

## Основна идея
Всеки ученик работи върху **собствения си дипломен проект** и изпълнява поредица от задания, които проследяват основните етапи на софтуерната разработка:

1. Анализ и планиране  
2. Проектиране и архитектура  
3. Реализация и организация на работата  
4. Тестване, документация и защита  

## Цел
Да се оцени не само крайният резултат, а и **процесът на разработка**:
- формулиране на изисквания;
- проектиране на решение;
- организирана работа с Git;
- реален напредък по проекта;
- тестване;
- документация;
- представяне и защита.

## Какво трябва да направи всеки ученик
Всеки ученик трябва да поддържа:
- свое GitHub repository за дипломния проект;
- папка `docs/`, в която качва изисканите `.md` файлове;
- актуален `README.md` в проекта;
- commit history, показваща реална работа по проекта.

## Файлове в това repository
- `task-01-analysis.md` – Задание 1
- `task-02-design.md` – Задание 2
- `task-03-implementation.md` – Задание 3
- `task-04-testing.md` – Задание 4
- `grading-rubric.md` – обща система за оценяване
- `submission-rules.md` – правила за предаване
- `templates/` – шаблони за ученическите документи

## Препоръчителна структура в ученическото repository
```text
project-root/
│
├── README.md
├── src/ или основните файлове на проекта
├── docs/
│   ├── 01-project-analysis.md
│   ├── 02-design-and-architecture.md
│   ├── 03-implementation-report.md
│   └── 04-testing-and-documentation.md
└── assets/ (по желание)
```

## Важно
Оценяването се извършва по **еднакви критерии за всички ученици**, независимо че работят по различни проекти.
