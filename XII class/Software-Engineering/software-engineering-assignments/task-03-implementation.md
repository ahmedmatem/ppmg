# Задание 3: Реализация, Git и качество на кода

## Цел
Да покажете реален напредък по дипломния проект, организирана работа и добри практики при разработката.

## Какво трябва да направите
Реализирайте **съществен функционален пакет** от проекта си.

Подходящи примери:
- регистрация и вход;
- CRUD модул;
- търсене и филтриране;
- управление на потребителски роли;
- резервации / заявки / поръчки;
- календар / график;
- качване и визуализиране на данни;
- друг основен модул според проекта.

## Какво трябва да подготвите
Създайте файл:

`docs/03-implementation-report.md`

В него включете:

### 1. Реализирана функционалност
Опишете какво конкретно сте добавили.

### 2. Как работи
Опишете основния поток на работа.

### 3. Възникнали проблеми
Опишете **поне 2 или 3 реални затруднения**, срещнати по време на разработката.

### 4. Как са решени
Опишете как сте ги преодолели.

### 5. Следващи стъпки
Опишете какво остава за довършване.

## Git изисквания
Необходимо е:
- поне **5 смислени commit-а**
- commit съобщенията да са описателни
- да има видим прогрес по задачата
- кодът да не бъде качен наведнъж без междинни стъпки

## Изисквания към кода
Ще се оценяват:
- четими имена;
- добра структура;
- логично разделяне на файлове / класове / модули;
- липса на хаотично копиране;
- базова четимост и подреденост.

## Изисквания за предаване
Ученикът трябва да предаде:
- реално качен код в repository
- попълнен файл `docs/03-implementation-report.md`
- обновен `README.md`
- screenshots или кратко видео по желание

## Критерии за оценяване (40 т.)
- реализирана функционалност – 15 т.
- работоспособност – 8 т.
- организация на кода – 6 т.
- Git дисциплина / commit history – 5 т.
- ясен отчет – 4 т.
- професионално предаване – 2 т.

## Срок
Попълва се от преподавателя.
