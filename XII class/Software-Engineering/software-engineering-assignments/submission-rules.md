# Правила за предаване

## 1. Общи правила
Всеки ученик работи върху **своя дипломен проект** и предава материалите по заданията в собствено GitHub repository.

## 2. Задължителни елементи
В ученическото repository трябва да присъстват:
- основният код на проекта;
- `README.md`;
- папка `docs/` с необходимите файлове;
- commit history, показваща реална работа.

## 3. Задължителни файлове в папка `docs/`
- `01-project-analysis.md`
- `02-design-and-architecture.md`
- `03-implementation-report.md`
- `04-testing-and-documentation.md`

## 4. Начин на предаване
Ученикът предава:
- линк към GitHub repository;
- при поискване – кратка демонстрация в час;
- при поискване – screenshot-и или кратко видео.

## 5. Изисквания към commit history
- commit-ите трябва да бъдат смислени;
- не се допуска формално качване на всичко наведнъж;
- commit съобщенията трябва да бъдат описателни;
- историята трябва да показва реален прогрес.

## 6. Изисквания към README
README файлът трябва да съдържа:
- име на проекта;
- цел;
- използвани технологии;
- основни функции;
- инструкции за стартиране;
- кратко описание на употребата.

## 7. Проверка на авторството
Преподавателят има право да изиска:
- кратко устно обяснение на кода;
- демонстрация на конкретна функционалност;
- обяснение на използваните технологии;
- обяснение на даден commit или файл.

## 8. При съмнение за несамостоятелна работа
Ако ученикът не може да обясни проекта си, commit history не показва реална работа или има несъответствия между предаденото и демонстрираното, преподавателят може да изиска допълнителна проверка или защита.
